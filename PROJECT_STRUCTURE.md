# 프로젝트 파일 구조 및 설명 (`dev_final`)

이 문서는 `dev_final` 프로젝트의 주요 파일 및 디렉토리 구조와 각 파일의 역할을 정리한 것입니다.

## 📁 프로젝트 루트 (Root)

### 백엔드 및 메인
| 파일명 | 설명 |
| :--- | :--- |
| **`main.py`** | **백엔드 서버**. FastAPI 프레임워크를 사용하여 API 서버를 구동합니다. 로그 저장, 회원 관리, 공연 정보 관리 등의 핵심 기능을 담당합니다. |
| `requirements.txt` | Python 의존성 라이브러리 목록 파일입니다. |

### 관리 및 분석 도구
| 파일명 | 설명 |
| :--- | :--- |
| **`viewer.html`** | **통합 로그 뷰어**. 사용자의 마우스 궤적, 클릭 로그, 호버 미세 떨림(매크로 탐지용) 등을 시각화하여 분석하는 관리자용 페이지입니다. |
| `viewer_performance.html` | (구) 공연 단계 전용 로그 뷰어. |
| `viewer_booking.html` | (구) 예매 단계 전용 로그 뷰어. |
| `admin.html` | 관리자 대시보드. 공연 정보(오픈 시간, 상태 등)를 추가, 수정, 삭제할 수 있습니다. |

### 사용자 웹 페이지 (프론트엔드)
| 파일명 | 설명 |
| :--- | :--- |
| **`index.html`** | **메인 페이지**. 공연 목록을 보여주고 사용자가 예매 여정을 시작하는 진입점입니다. |
| `performance_detail.html` | 공연 상세 페이지. 공연 정보를 확인하고 날짜 및 회차를 선택합니다. |
| `section_select.html` | 구역 선택 페이지. 좌석 등급 및 구역을 선택합니다. |
| **`seat_select.html`** | **좌석 선택 페이지**. 구역 내 세부 좌석을 지정합니다. 티켓 오픈 시간 카운트다운 및 자동 활성화 기능이 포함되어 있습니다. |
| `discount.html` | 할인 선택 페이지. 쿠폰이나 할인을 적용합니다. |
| `order_info.html` | 예매자 정보 입력 페이지. 주문자 정보와 배송지를 입력합니다. |
| `payment.html` | 결제 페이지. 결제 수단을 선택하고 최종 결제를 진행합니다. |
| `booking_complete.html` | 예매 완료 페이지. |
| `login.html` | 로그인 페이지. |
| `signup.html` | 회원가입 페이지. |
| `queue.html` | 대기열 안내 페이지. |

### 문서
| 파일명 | 설명 |
| :--- | :--- |
| `README.md` | 프로젝트 개요 및 실행 방법에 대한 메인 문서입니다. |
| `ACCESS_LINKS.md` | 주요 페이지 접속 링크 모음입니다. |
| `TICKET_OPEN_TIME_GUIDE.md` | 티켓 오픈 시간 설정 및 테스트 가이드 문서입니다. |
| `OPEN_TIME_FEATURE_SUMMARY.md` | 오픈 시간 기능 구현에 대한 상세 요약 문서입니다. |
| `ADMIN_ACCESS_GUIDE.md` | 관리자 기능 접속 및 사용 가이드입니다. |

## 📁 js (자바스크립트)

프론트엔드 로직을 담당하는 스크립트 파일들입니다.

| 파일명 | 설명 |
| :--- | :--- |
| **`logger.js`** | **핵심 로깅 모듈**. 마우스 움직임(trajectory), 클릭, 사용자 행동을 수집하여 서버 API로 전송합니다. |
| **`flow-manager.js`** | 예매 흐름 관리 모듈. 고유 `flow_id`를 생성/유지하고 사용자의 예매 단계(`flow_step`)를 추적합니다. |
| **`seat_select.js`** | **좌석 선택 로직**. 좌석 맵 렌더링, 좌석 선택/해제, **오픈 시간 카운트다운 및 새로고침 없는 자동 활성화(AJAX)** 로직이 핵심입니다. |
| `performance_detail.js` | 공연 상세 페이지 로직. 날짜/회차 선택 및 캘린더 기능을 처리합니다. |
| `admin.js` | 관리자 페이지 기능. API를 통해 공연 정보를 CRUD(생성/조회/수정/삭제) 합니다. |
| `auth.js` | 인증 유틸리티. 로그인 상태 확인, 토큰 관리, 비로그인 시 리디렉션 등을 처리합니다. |
| `utils.js` | 공통 유틸리티 함수 모음 (날짜 포맷팅 등). |
| `browser-info.js` | 브라우저 및 사용자 환경 정보 수집 스크립트. |
| `index.js`, `login.js`, `signup.js` | 각 해당 페이지의 전용 동작을 제어하는 스크립트입니다. |
| `section_select.js`, `discount.js` | 각 예매 단계 페이지의 로직을 담당합니다. |
| `order_info.js`, `payment.js` | 예매자 정보 입력 및 결제 처리 로직입니다. |

## 📁 css (스타일)

웹 페이지의 디자인과 레이아웃을 담당하는 스타일시트입니다.

| 파일명 | 설명 |
| :--- | :--- |
| **`main.css`** | **전역 스타일**. 기본 레이아웃, 폰트, `root` 변수(색상 테마 등)가 정의되어 있습니다. |
| **`components.css`** | **공통 컴포넌트**. 버튼, 카드, 입력 폼, 모달 등 재사용 가능한 UI 요소의 스타일입니다. |
| `admin.css` | 관리자 페이지 전용 스타일. |
| `seat_select.css` | 좌석 선택 페이지 전용 스타일. 좌석 배치도(Grid), 좌석 등급별 색상, 미니맵 등을 정의합니다. |

---

## 📁 data (데이터 저장소)

백엔드에서 사용하는 로컬 JSON 기반 데이터베이스입니다.

| 파일명 | 설명 |
| :--- | :--- |
| **`performances.json`** | **공연 정보 DB**. 공연 ID, 제목, 날짜, **티켓 오픈 시간(`open_time`)**, 상태(`upcoming`, `active` 등) 정보를 저장합니다. |
| `users.json` | 사용자 DB. 회원가입된 사용자 정보(이메일, 비밀번호, 이름 등)를 저장합니다. |

---

## 📁 logs (로그 저장소)

*   `logs/` 디렉토리에는 사용자가 예매 과정을 진행하면서 생성된 행동 로그 파일(`*.json`)들이 저장됩니다.
*   파일명 형식: `[날짜]_[공연ID]_[flow_id]_[성공여부].json`
*   `viewer.html`은 이 디렉토리의 파일들을 읽어들여 시각적으로 분석합니다.

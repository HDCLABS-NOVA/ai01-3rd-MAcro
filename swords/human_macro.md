# 🎫 티켓 예매 매크로 시스템 사양서 (Human-like Automation)

본 문서는 `swords` 패키지 내 주요 자동화 스크립트의 역할, 주요 기능 및 기술 스택을 기술합니다. 이 시스템은 단순 반복 매크로를 넘어, 인간의 행동 패턴을 모방하고 시각 지능(VLM)을 활용하여 탐지를 회피하는 차세대 자동화 기술을 포함합니다.

---

## 1. `ticket_booking_automation.py`: 브라우저 기반 엔드 투 엔드 자동화

### 📋 역할
티켓 예매 사이트의 접속부터 최종 예매 완료까지의 전체 프로세스를 관리하는 메인 자동화 엔진입니다.

### ✨ 주요 기능
1. **전 과정 자동화 (End-to-End)**: 로그인, 공연 선택(랜덤), 일시 선택, 좌석 선택, 최종 결제 단계까지 연속적인 워크플로우를 수행합니다.
2. **인간 행동 모방 (Human-like Emulation)**:
    - **자연스러운 마우스 이동**: 고정된 좌표가 아닌, 베지에 곡선(Bezier Curve) 기반의 곡선 경로와 가속/감속을 적용하여 사람의 마우스 조작을 모방합니다.
    - **무작위 지연 시간**: 각 단계 사이의 대기 시간을 고정값이 아닌 일정 범위 내의 확률적 지연 시간(Random Jitter)으로 설정합니다.
    - **동적 스크롤**: 클릭 타겟이 화면 밖에 있을 경우, 자연스럽게 스크롤하여 시야를 확보한 후 상호작용합니다.
3. **환경 탄력적 설계**: 
    - URL 파라미터 기반 접속 (localhost, ngrok 등 도메인 제약 없음).
    - 서버 환경에 따른 경로(예: `/html/` 접두사 여부) 자동 감지 및 보정.
4. **통계 및 로깅**: 모든 시도의 성공/실패 여부와 단계별 걸린 시간을 기록하고 정밀한 에러 스크린샷 및 HTML 덤프를 생성합니다.

### 🛠 사용 기술
- **Language**: Python 3.x
- **Automation**: Playwright (Sync API)
- **Geometry**: Bezier Curve 모듈을 통한 마우스 궤적 생성
- **Logging**: Python logging, JSON stats 기반 사후 분석 시스템

## 2. `ticketpark_auto.py`: VLM 기반 지능형 모니터링 및 팝업 처리

### 📋 역할
브라우저 외부 환경이나 복잡한 이미지 기반 CAPTCHA, 팝업창을 시각 지능(VLM)을 통해 분석하고 시스템 레벨에서 처리하는 '지능형 와처(Watcher)'입니다.

### ✨ 주요 기능
1. **VLM 기반 분석**: LM Studio 또는 Groq의 시각 언어 모델(Vision Language Model)을 호출하여 현재 화면 상태를 인간처럼 이해합니다.
2. **CAPTCHA 자동 해결**: 
    - 화면의 왜곡된 문자를 인식하여 텍스트로 추출합니다.
    - 인식이 어렵거나 중복된 패턴인 경우 자동으로 '새로고침' 버튼을 눌러 재시도합니다.
3. **시스템 레벨 상호작용 (OS-level)**: 
    - 브라우저 제어 권한을 넘어서는 팝업이나 시스템 경고창을 `PyAutoGUI`를 통해 직접 클릭하고 키보드 입력을 전달합니다.
4. **GUI 제어 패널**: 
    - 사용자 친화적인 Tkinter 기반 인터페이스를 제공하여 실시간 로그 확인 및 옵션(사용 모델, 자동 처리 대상 등)을 제어할 수 있습니다.
5. **다중 트리거 감지**: 특정 공연 아이콘, 날짜/시간 선택 창, 타임아웃 경고 등 다양한 상태를 동시에 감지하고 우선순위에 따라 응답합니다.

### 🛠 사용 기술
- **Language**: Python 3.x
- **Vision**: VLM (Vision-Language Models via API)
- **GUI**: Tkinter (Dark Theme 적용)
- **System Control**: PyAutoGUI, Pillow (Screen Capture)
- **Concurrency**: Threading (백그라운드 감지 루프)

## 3. 시스템 연동 아키텍처 (Synergy)

| 구분 | `ticket_booking_automation.py` | `ticketpark_auto.py` |
| :--- | :--- | :--- |
| **주요 영역** | 브라우저 DOM 객체 제어 | 전체 화면 이미지 분석 (VLM) |
| **강점** | 복잡한 데이터 플로우 및 속도 | CAPTCHA 해결 및 유연한 상황 대처 |
| **상호 보완** | 웹 구조 기반의 안정적인 예매 | 웹 구조가 바뀔 때 시각 정보로 대응 |

이 두 스크립트의 유기적인 결합을 통해, 보안 솔루션이 강화된 최근 티켓 예매 환경에서도 높은 성공률과 탐지 우회 가능성을 유지하도록 설계되었습니다.

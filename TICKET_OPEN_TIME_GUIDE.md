# 🎫 티켓 오픈 시간 관리 시스템 사용 가이드

## 📌 개요

티켓 예매 사이트에서 공연별로 티켓 판매 오픈 시간을 설정하고 관리할 수 있는 관리자 전용 대시보드입니다.

**✨ 예매 페이지와 완전 연동!**
- 관리자가 설정한 오픈 시간을 실시간으로 예매 페이지에 반영
- 오픈 전에는 예매 버튼 비활성화 및 카운트다운 표시
- 오픈 시간 도달 시 자동으로 예매 가능 상태로 전환

## 🎯 주요 기능

### 1️⃣ 공연 목록 조회

대시보드에 접속하면 등록된 모든 공연의 카드가 표시됩니다.

각 카드에는 다음 정보가 표시됩니다:
- **공연명**: 공연 제목
- **카테고리**: concert, musical, sports, exhibition 등
- **장소**: 공연 장소
- **공연 ID**: 고유 식별자
- **상태**: 오픈 예정 / 판매중 / 판매 종료
- **티켓 오픈 시간**: 현재 설정된 오픈 시간

### 2️⃣ 오픈 시간 설정/수정

각 공연 카드에서:

1. **날짜/시간 선택 필드** 클릭
2. 원하는 **티켓 오픈 날짜와 시간** 선택
3. **💾 저장** 버튼 클릭

> 💡 시간대는 자동으로 한국 시간(KST, UTC+9)으로 처리됩니다.

**예시:**
```
오픈 시간: 2026-02-10 14:00
→ 2026년 2월 10일 오후 2시에 티켓 판매 시작
```

### 3️⃣ 실시간 카운트다운

오픈 시간이 **미래 시간**으로 설정된 경우:
- 자동으로 **실시간 카운트다운 타이머**가 표시됩니다
- 형식: `⏰ 오픈까지: X일 X시간 X분 X초`

**특별 효과:**
- 오픈까지 **1시간 미만** 남았을 때: 타이머가 빨간색으로 강조됩니다
- 오픈 시간 도달: `🎉 판매 오픈!` 메시지 표시

### 4️⃣ 상세 정보 확인

각 카드의 **👁️ 상세** 버튼을 클릭하면:
- 해당 공연의 상세 정보 확인 가능 (추후 확장 예정)

## 📊 사용 예시

### 예시 1: 신규 공연 오픈 시간 설정

```

2. 2026-02-10 20:00 선택
3. 💾 저장 버튼 클릭
4. "✅ 오픈 시간이 업데이트되었습니다!" 알림
```

## 🛠️ 기술 세부사항

### API 엔드포인트

관리자 대시보드는 다음 API를 사용합니다:

#### 1. 공연 목록 조회
```http
GET /api/performances
```

**응답 예시:**
```json
{
  "success": true,
  "count": 6,
  "performances": [
    {
      "id": "perf001",
      "title": "아이유 콘서트 <The Golden Hour>",
      "category": "concert",
      "venue": "고척스카이돔",
      "open_time": "2026-02-10T14:00:00+09:00",
      "status": "upcoming"
    }
  ]
}
```

#### 2. 오픈 시간 업데이트
```http
PUT /api/admin/performances/{performance_id}
Content-Type: application/json

{
  "open_time": "2026-02-10T14:00:00+09:00",
  "status": "upcoming"
}
```

**응답 예시:**
```json
{
  "success": true,
  "message": "공연 정보가 업데이트되었습니다.",
  "performance": { ... }
}
```

### 데이터 구조

`data/performances.json` 파일에 저장됩니다:

```json
{
  "performances": [
    {
      "id": "perf001",
      "title": "아이유 콘서트 <The Golden Hour>",
      "category": "concert",
      "venue": "고척스카이돔",
      "dates": ["2026-02-15", "2026-02-16", "2026-02-17"],
      "times": ["18:00"],
      "grades": [...],
      "image": "images/concert01.jpg",
      "description": "아이유의 2026년 첫 단독 콘서트",
      "open_time": "2026-02-10T14:00:00+09:00",
      "status": "upcoming"
    }
  ]
}
```

### 상태 값 (status)

- **`upcoming`**: 오픈 예정 (기본값)
- **`open`**: 판매중
- **`closed`**: 판매 종료

> 현재 버전에서는 자동 상태 변경이 구현되지 않았습니다. 필요시 수동으로 업데이트해야 합니다.

## 🔐 보안

### 권한 체크
- `js/auth.js`의 `requireAdmin()` 함수로 클라이언트 측 검증
- 관리자 이메일: `admin@ticket.com`, `manager@ticket.com`

### 프로덕션 환경 권장사항
현재는 개발/데모 용도입니다. 실제 프로덕션에서는:

1. **서버 측 권한 검증** 추가
2. **JWT 토큰 기반 인증** 구현
3. **비밀번호 해시화** (bcrypt)
4. **환경 변수**로 관리자 계정 관리
5. **감사 로그** (누가, 언제, 무엇을 변경했는지)

## ❓ FAQ

### Q1. 오픈 시간을 과거로 설정하면?
A. 설정은 가능하지만, 카운트다운이 표시되지 않고 `🎉 판매 오픈!` 메시지가 즉시 표시됩니다.

### Q2. 여러 공연의 오픈 시간을 동시에 설정할 수 있나요?
A. 현재는 각 공연마다 개별적으로 저장해야 합니다.

### Q3. 오픈 시간이 되면 자동으로 상태가 변경되나요?
A. 현재 버전에서는 자동 변경되지 않습니다. 추후 백그라운드 작업으로 구현 가능합니다.

### Q4. 새 공연을 추가하려면?
A. 현재는 `data/performances.json` 파일을 직접 수정하거나, API를 통해 추가해야 합니다. UI에서의 추가 기능은 추후 구현 예정입니다.

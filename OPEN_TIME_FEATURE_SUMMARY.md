## 🐛 해결된 버그

### 버그 1: Timezone 변환 오류
**문제**: 관리자가 10:25를 설정해도 01:25 (새벽 1시)로 저장됨  
**원인**: `.toISOString()`이 UTC로 변환하면서 9시간 차감  
**해결**: 직접 문자열 조합 (`newOpenTime + ':00+09:00'`)

### 버그 2: JavaScript 함수 직접 호출 가능
**문제**: disabled만으로는 콘솔에서 `selectDate()` 호출 가능  
**원인**: 함수가 항상 실행 가능한 상태  
**해결**: `isCurrentlyOpen()` 체크 로직 추가

### 버그 3: onclick 속성 우회
**문제**: 개발자 도구로 disabled 제거 후 클릭하면 실행됨  
**원인**: inline `onclick="selectDate()"` 사용  
**해결**: onclick 제거 → 이벤트 리스너로 변경 (오픈 후에만 등록)

## 📊 기술 세부사항

### 시간대 처리
- **서버 시간**: 한국 시간 (KST, UTC+9)
- **형식**: ISO 8601 (`2026-02-03T11:15:00+09:00`)
- **입력**: datetime-local (`2026-02-03T11:15`)
- **출력**: 한국식 포맷 (`2026년 2월 3일 (월) 오전 11:15`)

### 카운트다운 로직
- **업데이트 주기**: 1초 (`setInterval(fn, 1000)`)
- **표시 형식**: `X일 X시간 X분 X초`
- **1시간 미만**: 빨간색 + 펄스 애니메이션
- **오픈 도달**: 초록색 + "🎉 판매 시작!"
- **동적 활성화**: 1초 후 `activateBookingPage()` 실행

### 상태 관리
- **`upcoming`**: 오픈 예정 (빨간 배지)
- **`open`**: 판매중 (초록 배지)
- **`closed`**: 판매 종료 (회색 배지)

## 🐛 해결된 버그

### 버그 1: Timezone 변환 오류
**문제**: 관리자가 10:25를 설정해도 01:25 (새벽 1시)로 저장됨  
**원인**: `.toISOString()`이 UTC로 변환하면서 9시간 차감  
**해결**: 직접 문자열 조합 (`newOpenTime + ':00+09:00'`)

## 📊 기술 세부사항

### 시간대 처리
- **서버 시간**: 한국 시간 (KST, UTC+9)
- **형식**: ISO 8601 (`2026-02-03T10:26:00+09:00`)
- **입력**: datetime-local (`2026-02-03T10:26`)
- **출력**: 한국식 포맷 (`2026년 2월 3일 (월) 오전 10:26`)

### 카운트다운 로직
- **업데이트 주기**: 1초 (`setInterval(fn, 1000)`)
- **표시 형식**: `X일 X시간 X분 X초`
- **1시간 미만**: 빨간색 + 펄스 애니메이션
- **오픈 도달**: 초록색 + "🎉 판매 시작!"

### 상태 관리
- **`upcoming`**: 오픈 예정 (빨간 배지)
- **`open`**: 판매중 (초록 배지)
- **`closed`**: 판매 종료 (회색 배지)

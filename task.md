# 🛡️ 티켓팅 매크로 탐지 시스템 로그 수집 프로젝트 (Task Summary)

## 📌 프로젝트 개요
본 프로젝트의 목적은 티켓팅 예매 과정에서 발생되는 사용자 행동 로그(마우스 궤적, 클릭 시간, 호버 패턴 등)를 수집하고 분석하여, 사람과 매크로를 정확히 구분하는 머신러닝 탐지 모델을 구축하는 것입니다.

---

## 🛠️ 주요 작업 내역

### 1. 로깅 시스템 고도화 (`js/logger.js`)
매크로와 사람의 물리적 차이를 분석하기 위한 정밀 로그 수집 기능을 구현했습니다.
*   **클릭 유지 시간 (`click_duration`)**: `mousedown`부터 `mouseup`까지의 밀리초(ms)를 측정하여 기계적 클릭(1~2ms)과 인간적 클릭(50~150ms)을 구분.
*   **실제 호버 기록 (`hovers`)**: 마우스가 좌석 위에 머무는 행위를 기록하여 '탐색 의도'를 데이터화.
*   **정규화된 좌표 수집**: 해상도에 관계없이 분석 가능하도록 0~1 사이의 정규화 좌표(`nx`, `ny`) 및 정수 좌표 여부(`is_integer`) 기록.
*   **자동 로그 분류**: `sessionStorage`의 `bot_type`을 읽어 서버에서 `logs/macro/`와 `logs/human/`으로 자동 분류하도록 구현.
*   **데이터 유실 방지**: 페이지 이동 시 기존에 수집된 `clicks`와 `hovers` 배열이 덮어씌워지는 버그 수정.

### 2. 매크로 봇 고도화 (`automation/macro_bot.js`)
가장 진보된 형태의 픽셀 기반 매크로를 시뮬레이션합니다.
*   **초고속 타격**: 1ms 수준의 클릭 시간과 직선형 순간이동 경로 생성.
*   **지능형 우회 (Stealth Mode)**: 30%의 확률로 클릭 유지 시간을 사람과 비슷한 30~80ms로 랜덤하게 설정하여 모델의 판별력을 테스트.
*   **반복 수집 기능**: 100~300회 이상의 대량 루프 및 백그라운드(`headless: true`) 실행 지원.

### 3. 휴먼 봇 고도화 (`automation/human_bot.js`)
실제 예매 현장에서의 사람의 긴박한 움직임을 시뮬레이션합니다.
*   **연속적 이동 궤적**: `page.mouse.move`의 `steps` 옵션을 사용하여 가느다란 비선형 이동 경로 생성.
*   **물리적 클릭 지연**: 최소 60~100ms의 클릭 유지 시간을 보장하여 기계적 신호와 차별화.
*   **즉각적 타격**: 정보를 훑는 딜레이는 최소화하되, 이동 과정에서의 '인간미'를 궤적 데이터로 남김.

### 4. 서버 및 분석 환경 (`main.py`, `viewer.html`)
*   **서버**: 수신된 로그의 메타데이터를 분석하여 매크로/휴먼 계정별로 별도 디렉터리에 JSON 파일 저장.
*   **뷰어**: 수집된 로그의 클릭 지속시간(ms)을 툴팁으로 표시하고, 마우스 궤적 및 호버 빈틈(Tremor)을 시각적으로 분석하는 기능 제공.
